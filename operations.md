## Day-2 operations

## Health and observability

Track these key signals continuously:

- CPU, memory, storage, and thermal headroom
- Minima node reachability and sync status
- DApp host stability and crash loops
- Network tunnel status and peer health

## Update workflow

- Validate release notes and dependency impact
- Snapshot critical configuration before change
- Roll updates in controlled windows
- Re-verify services, DApps, and node APIs post-update

## Backup and recovery

- Back up configuration and DApp manifests regularly
- Keep boot profiles/snapshots for rollback
- Test restore drills before production usage

## Cluster operations

- Maintain node inventory and role assignments
- Enforce secure mesh links between nodes
- Monitor consistency/state convergence across the cluster

## Operational safeguards

- Principle of least privilege for operators
- Restricted access paths to management APIs
- Auditable change process for system-level configuration
